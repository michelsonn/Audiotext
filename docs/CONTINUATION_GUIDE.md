# Audiotext — руководство для продолжения разработки

Этот файл предназначен для нового чата, другого разработчика или модели, которой потребуется продолжить проект без потери контекста.

## 1. Текущее состояние

Рабочая версия: **1.0.6**.

Программа уже реально запускается на Windows, скачивает модель и успешно транскрибирует аудио на NVIDIA GPU. Основной движок менять без необходимости нельзя: он проверен на реальных звонках.

## 2. Критические файлы

```text
main.pyw
RUN_DEV.bat
BUILD_EXE.bat
Audiotext.spec
version_info.txt
requirements.txt
requirements-build.txt

audiotext\
  __init__.py
  app.py
  engine.py

resources\
  Audiotext.ico
  spin_up.svg
  spin_down.svg

docs\
  SPECIFICATION.md
  DEVELOPMENT_HISTORY.md
  CONTINUATION_GUIDE.md
```

- `engine.py` — транскрибация и выходные форматы;
- `app.py` — GUI, очередь, настройки, модель, пути данных;
- `__init__.py` — версия;
- `Audiotext.spec` — сборка PyInstaller.

## 3. Правила безопасных изменений

1. Перед патчем брать последнюю рабочую версию `app.py`, а не исходную beta.
2. Патч должен быть накопительным относительно последней версии.
3. Не менять `engine.py`, если правка касается только интерфейса.
4. После изменений обязательно выполнять:

```bash
python -m py_compile audiotext/app.py audiotext/engine.py
```

5. На Windows проверить:
   - запуск `RUN_DEV.bat`;
   - открытие настроек;
   - загрузку модели;
   - один короткий файл;
   - несколько воркеров;
   - закрытие через трей.
6. Перед заменой пользователь должен полностью закрыть приложение, включая значок в трее.
7. Не удалять `models` и `data` при обновлении.

## 4. Важные архитектурные решения

- portable-first: данные рядом с EXE;
- модель не встраивается в EXE;
- EXE распространяется папкой;
- NVIDIA/CUDA only;
- friendly label модели отделён от технического ID;
- локальная папка модели имеет приоритет;
- JSON включён по умолчанию, но считается служебным;
- пользовательские результаты никогда не должны храниться внутри временной папки PyInstaller `_MEIPASS`.

## 6. Особое замечание про Program Files

Текущая portable-архитектура пишет `data` и `models` рядом с EXE. В `C:\Program Files` обычный пользователь может не иметь права записи. Поэтому будущий установщик должен выбрать одну из стратегий:

1. устанавливать Audiotext в пользовательскую папку, где есть права записи; либо
2. при установленной версии хранить `data/models` в `%LOCALAPPDATA%`, сохранив portable-режим для ZIP-сборки; либо
3. позволить пользователю выбрать отдельную папку моделей и данных.

До создания установщика менять portable-логику не требуется.
