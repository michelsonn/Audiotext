# Участие в разработке Audiotext

Спасибо за интерес к проекту.

## Перед началом

1. Создайте issue с описанием ошибки или предложения.
2. Для исправления создайте отдельную ветку от актуальной основной ветки.
3. Не добавляйте модели, пользовательские аудио, расшифровки, настройки, виртуальные окружения и результаты сборки.

## Локальная проверка

```bat
RUN_DEV.bat
```

Минимальная проверка синтаксиса:

```bat
python -m compileall -q audiotext main.pyw tests
```

Тесты:

```bat
python -m unittest discover -s tests -v
```

## Pull request

В описании укажите:

- что изменено;
- зачем это нужно;
- как проверялось;
- затрагивается ли совместимость настроек, очереди и структуры результатов;
- есть ли изменения в сторонних зависимостях.

Старайтесь делать небольшие изолированные изменения. Не смешивайте косметический рефакторинг с исправлением функциональной ошибки.

## Лицензия вкладов

Отправляя вклад в проект, вы соглашаетесь распространять его по лицензии MIT проекта Audiotext.
